B. 向量语义检索（⽇记 / 知识库）
这是 pgvector 的核⼼⽤途，我们⾃⼰实现的 RAG：
SELECT content, 1 - (embedding <=> $query::vector) AS similarity
FROM knowledge_chunks WHERE room = $room
ORDER BY embedding <=> $query::vector LIMIT 5 -- 余弦距离，取相似度 > 0.3

六、⽇记重抽审阅合并（redigest）

embedding 语义配对新旧记忆，三组并排审阅：

- 仅新（新增的）：默认添加
确认后才事务性落库（删选中旧 + 插选中新）

多处防丢兜底：孤⼉默认保留、pinned 旧记忆默认保留
设计要点：「预览只读 → ⼈⼯逐条审阅 → 确认才写库」。取代了早期「先删后抽再插」的破坏性流程——
那种流程⼀旦抽取失败就会丢记忆。
七、记忆整理：脱⽔ · 合并 · 相似聚类
记忆库会越积越多——有的写得太⻓注⽔，有的内容彼此重复。这⼀套基于选择的批量整理应对它，并⽤向量
聚类帮你发现该整理的。
选择模式（统⼀⼊⼝）
列表「选择」开关 → 勾选 N 条 → 底部栏先选模型（DeepSeek V4 Flash 默认 · Sonnet 4.5 · V4 Pro）→
点 [脱⽔] 或 [合并]。脱⽔与合并共⽤同⼀套选择 + 模型前置（执⾏前就选好，不是事后才换）。
脱⽔（dehydrate）— 压短，不丢温度
把注⽔的记忆拧⼲，情感质感与独特细节⼀字不损：
硬约束：① 结果必须⽐原⽂短（变⻓ / 没压缩 → 弃⽤重试，两次都不达标就标「⽆变化」不动它）；② 锁
死原⽂语⾔（检测假名漏译，中⽂记忆不会被悄悄改写成⽇语）
逐条预览：原⽂ ↔ 脱⽔后并排，标字数与压缩百分⽐
版本对⽐：「再脱⽔」追加候选 v1 / v2…，切换⽐较，挑最满意的那版采⽤
每条可⼿动微调，勾选后批量写回
合并（merge）— 多条糅成⼀条
把⼏条相近记忆融合：重复的归⼀、各⾃独特的全保留、⾃然糅合、保温度地精简：
多条原⽂并排 + 合并结果（可编辑）
版本对⽐：「再合并」追加候选，切换选择
采⽤后：新记忆⼊库，原 N 条归档（不删除，留安全⽹；任⼀原条置顶则新条继承置顶）
相似聚类排序（類似順）— 发现重复
（text-embedding-3-small，1536 维）+ 余弦单链接聚类
memories.embedding
切到「類似順」→ 语义相近的记忆聚成组、相邻排列，打上 「🔗 似ている記憶 ×N」 分组标头
⼀眼看出哪⼏条讲的是⼀回事 → 直接在那组⾥勾选、合并（发现 → 合并动线闭环）
惰性维护：改过 content 的记忆向量置空，下次⽤时⾃动补⻬（⾸次切⼊即全量回填）
脱⽔ / 合并的 LLM 调⽤⾛ provider 路由（DeepSeek / Anthropic），带 429 限流退避重试 + 低并发，
避免批量整理打爆 API 速率上限。
